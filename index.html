🚀 How to Use EMELCO Trip Manager

1
Your data is already loaded
628 trips from your CSV have been pre-loaded into the app. They are saved in your browser's local storage — they stay even after closing the browser tab or the file.
2
Adding a new trip daily
Click "+ New Trip" in the top right. Fill in the required fields (Trip Date, Client, Driver, Plate). Formula fields like Rate+Surcharge, Income, %Profit, and Week numbers compute automatically as you type.
3
Editing a trip
Find the trip in the table and click the ✏️ button on the right. Update any field. All formula fields recalculate live.
4
Managing Staff (Drivers & Helpers)
Go to the 👤 Staff tab to view all drivers and helpers. Click Edit on any card to update their name, start date, status (Active/Idle), or position. Add new staff with the "+ Add Staff" button. Staff changes immediately reflect in the trip entry dropdowns.
5
Managing the Fleet
Go to the 🚛 Fleet tab to view all trucks with their plates, types, and usage stats. Add or edit trucks as needed. Fleet changes appear immediately in the truck dropdown when adding trips.
6
Filtering & Searching
Use the search box and dropdown filters (Client, Area, Week, Driver, Doc Status) to quickly find trips. All filters work together.
7
Exporting data back to Excel
Click "⬇ Export CSV" to download all trips with all computed columns. Open in Excel and the layout matches your original Trip List format.
8
Backup & Restore
Go to the ⬆ Data tab. Click "Backup Full Data (JSON)" regularly to save a copy. To move data to another PC or browser, backup on the old one, then use "Restore from JSON Backup" on the new one.
9
Where is data stored?
All data lives in your browser's localStorage for the HTML file. As long as you open the same HTML file in the same browser, your data persists. Do NOT clear browser data/cache for this file. Use JSON Backup to be safe.

📋 Data Entry Rules & CSV Import Guide

⚠️
No Driver / No Plate — Convoy & Reassignment Trips
Some trips legitimately have no driver or no plate assigned — e.g. a load was split across multiple trucks/drivers ("convoy"), or an assigned truck broke down and another truck picked up the load instead.

To save a trip like this: check the box "⚠️ No driver and/or no plate — convoy reassignment" in the New/Edit Trip form, and type a short reason in the box that appears (e.g. "PINAGHATIAN PO NG TATLONG TRUCK NA KA CONVOY"). The reason is required — the trip will not save without it.

The reason is automatically added to Gen Remarks, and the trip is flagged with an orange ⚠️ NO DRIVER / ⚠️ NO PLATE badge in the table so it's easy to spot for review during billing or audits.
🔑
Trip Code vs. TripKey — what's the difference?
Code is the main, human-readable ID for a trip (e.g. 260618CBP8680CMIPJas) — built from date + plate + client + driver initials when the trip is first created. It is your billing-facing reference and never changes automatically, even if you later correct the driver or plate on that trip.

TripKey is a separate, optional column used only to help CSV re-imports correctly match a row to the right existing trip — especially when the same plate runs more than once in a day, or when a trip spans multiple days. It is not used for billing and does not affect Code, Trip Ticket, or any financial figures.
🔢
Setting up TripKey in Excel
Add a TripKey column to your Excel Trip List. For most rows, this is auto-filled with a formula that counts repeat trips per plate per day:
=COUNTIFS($A$2:A2, A2, $C$2:C2, C2) (Adjust column letters to match your actual Date and Plate columns.) Drag this formula down the whole sheet — it numbers each repeat trip for the same plate on the same day as 1, 2, 3, etc. Single trips per day just get 1 automatically.

For trips spanning multiple days (redelivery, multi-day hauls): manually overwrite the TripKey on the later day's row with the same value as day 1 — using a letter (A, B, C…) instead of a number makes it visually clear it's a continuation, not a same-day repeat.
📤
How CSV re-import works (correcting past trips)
When you re-upload a CSV containing rows you've corrected (driver, rate, dates, remarks, etc.):
• A row whose Code already exists in the app will update that trip with the corrected values — it will not be skipped or duplicated.
• Diesel entries, multi-drop destinations, and the No-Driver/Plate flag entered inside the app are preserved during this update — they will not be wiped out by the CSV's blank columns for those fields.
• A row with a new Code not yet in the app will be added as a new trip, after a safety check for exact duplicates (same date+plate+driver+area+destination).
• The import result message will show you exactly how many trips were added vs. updated vs. skipped, so you can always verify what happened.
✅
Best practice checklist before re-importing
1. Make sure corrected rows keep their original Code unless the trip is genuinely a different trip.
2. Fill in TripKey for any plate that ran more than once on the same day, or for multi-day trips.
3. Check the import summary message after upload (added / updated / skipped counts) to confirm the result matches what you expected.
4. If something looks off, use the 🔴 Exact / 🟡 Similar duplicate filters on the Trips page to review and clean up any stray records.

🚚 Fleet & Staff Idle Tracking

⏱
Idle days, automatically tracked
Both the 👤 Staff and 🚛 Fleet tabs now show "Last Trip" and how many days idle each driver, helper, or truck has been — calculated automatically from their most recent trip record. No manual entry needed; it updates as soon as a new trip is saved. Idle counts of 7+ days are highlighted in red so they're easy to spot at a glance.
🔧
Marking a truck "Under Maintenance"
Open the truck in the 🚛 Fleet tab, click ✏️ Edit, and set Fleet Status to Maintenance. This is separate from "Idle" — Idle means the truck is available but not currently assigned a trip, while Maintenance means it's out of service and shouldn't be assigned trips until changed back to Active.
